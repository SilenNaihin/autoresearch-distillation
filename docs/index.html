Abstract

We train Qwen3-14B to propose modifications to a GPT pretraining script to reduce validation loss using Self-Distillation Policy Optimization (SDPO). Modified training scripts are run on H100 GPUs and natural language describing experimental outcomes is used as hindsight context for a teacher model which provides a reward signal to a student model. On Karpathy's autoresearch benchmark, SDPO achieves val_bpb = 1.028 during training — and 1.023 when the trained checkpoint is evaluated with ICL, a 3.1% improvement that surpasses the original 2.8% improvement from the Karpathy agent. We compare against two baselines — multi-turn in-context learning and single-turn random exploration — finding that context conditioning adds consistency without improving outcomes, while weight updates improve the model's capabilities, leading to improvements not produced by the base model.

Three Approaches

SDPO RL

The model generates rollouts — editing train.py via bash tool calls — and experiments execute on remote H100s. A reward signal is provided via hindsight context from a teacher model, flowing to the model's weights via policy gradient updates. SDPO does not need a successful demonstration for an advantage signal — it only needs the correct solution to be more likely under the teacher model compared to the student model, providing dense supervision even when all rollouts fail. Weight updates encode exploration patterns that compound across training — increasing effective memory and feedback adherence in comparison to context-bounded approaches (such as ICL and memory systems).

ICL Baseline In-Context

Same model, same tools, no weight updates. The model accumulates experiment feedback in its context window across 50 sequential turns: top results with full diffs, classified crash reports, and structured history. All learning happens through context conditioning. Best result: val_bpb = 1.038 across multiple runs.

Single-turn Baseline No Feedback

N independent single-shot calls with no history and no feedback. Just “here’s train.py, improve it” repeated N times, taking the best result. This measures the entropy of the sampling distribution — what you get from pure random exploration at temperature 1.0. At 50 turns, single-turn achieves val_bpb = 1.032 — actually beating ICL’s best of 1.038, suggesting the feedback loop adds consistency but not fundamentally better outcomes at this scale.

What the Model Finds

The dominant insight across both SDPO and ICL: trade model size for training throughput. Within the fixed 5-minute budget, a smaller model that completes more gradient steps outperforms a larger model that trains fewer steps. The best SDPO result combines ASPECT_RATIO 40, DEPTH 12, batch 64, and learning rate tuning — a compound edit that increases depth while reducing width and batch size, achieving 1.024 val_bpb.

Learned intuition through weight updates

The SDPO prompt contains zero experiment history — just the system prompt and train.py. Yet after training, the model generates chain-of-thought that references “previous attempts” and their outcomes. These aren’t memories from context; they’re learned intuitions encoded in the weights through SDPO, surfacing as recollections in chain-of-thought used to guide reasoning:

The model has never seen experiment feedback in its prompt. There is no “user” and no “previous attempt.” But through SDPO training on thousands of rollouts, the model’s weights have internalized which changes help, which crash, and why — all encoded as intuition, not context.

Weights vs. Context

The core question: does RL training add value beyond what in-context learning provides?

Single-turn beats ICL. The single-turn baseline (1.032) actually outperforms ICL (1.038) through a single lucky hit. The feedback loop gives ICL more consistent improvements, but doesn’t produce fundamentally better outcomes — random exploration at temperature 1.0 finds better configurations by chance. Context conditioning adds reliability, not capability.

SDPO breaks out. Weight updates encode exploration patterns that persist across rollouts without consuming context. The model learns which kinds of changes tend to work. For example, early in training the model attempts to increase the model size before finding that this leads to out-of-memory errors. After ~10 steps the model learns to avoid these modifications and explores other ways of improving validation loss (e.g., reducing aspect ratio and batch size). SDPO achieves 1.028, a clear separation from both baselines.

The gap between single-turn and ICL measures what context contributes (nothing — single-turn actually wins). The gap between both baselines and SDPO measures what weights contribute.

Comparison to the Karpathy Agent

The original autoresearch experiments used an LLM-based agent on H100 GPUs, running 126 experiments over multiple cycles. The best result improved val_bpb by 2.8% (0.998 → 0.970). Key improvements: batch halving for more gradient steps (−0.012), adding depth (−0.004), embedding LR tuning (−0.003), and weight decay adjustments.

A key design difference: Karpathy’s setup accepts successful diffs into train.py, so each experiment builds on the last improvement — the file evolves over time. In our setup, the agent always receives the original unmodified train.py. This is deliberate: SDPO rollouts start from scratch each time (learning lives in the weights, not the file), so the baselines must do the same for a fair comparison. It means every improvement must be independently rediscovered, making our results harder to achieve but cleaner to interpret.

Despite this handicap, our SDPO-trained checkpoint evaluated with ICL (−3.1%) surpasses the original Karpathy agent (−2.8%) using a 14B open-source model trained with RL. The dominant insight is the same across both: throughput over parameters — trade model size for more training steps within the fixed budget. Both agents independently converge on this strategy.

Limitations & Future Work

Our current approach depends on the base capabilities of a Qwen3 14B model in generating novel modifications to research code. We expect the performance of this approach to improve with the size of the model and with additional tools (e.g., web search to read research papers). Another challenge is encouraging the model to combine single edits that reduce validation loss. Finally, although we believe references to previous experiments in chain-of-thought serve as a means of improving reasoning quality, we notice the model tends to conflate feedback from different rollouts. We suspect this is due to using a batch size greater than one with the same prompt and varying teacher hindsight context.

Built On

Autoresearch (Karpathy) — A single-file GPT pretraining script with Muon+Adam optimization, FlashAttention 3, and sliding window attention. The benchmark: modify train.py to minimize val_bpb within a 5-minute training budget on a single H100.

SDPO (Hübotter et al., 2026) — Self-Distillation Policy Optimization. Converts tokenized environment feedback into a dense learning signal by treating the current model conditioned on feedback as a self-teacher, then distilling its corrected next-token predictions back into the policy. We extend it with agentic tool use and live environment rewards from real GPU experiments.
